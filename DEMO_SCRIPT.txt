[0:20 — WHAT IT IS]

* SPEAK:
  "Med-Companion is built for the 'Agents for Good' track. It focuses 
  on patient safety, data security, and keeping a human in control."


[0:40 — CORE PILLARS]

* VISUAL: (Point to the three icons on the cover banner)

* SPEAK:
  "As you can see on our project banner, the application rests on three

  Verification."

* ACTION: (Pause for 2 seconds)


[1:00 — HOW THE SYSTEM WORKS (ARCHITECTURE)]

* VISUAL: (Switch screen to show: assets/architecture_diagram.png)

* SPEAK:
  "Let's look at the workflow diagram to see how the system works.

  First, a user's question goes through our Security Checkpoint. This node 
  automatically checks for hacking attempts and hides personal info like 
  emails and Social Security Numbers.

  Once clean, the query goes to the main Orchestrator Agent.

  The Orchestrator then delegates tasks to specialized helper agents: 
  the safety_agent checks drug guidelines, and the scheduler_agent handles 
  medication logs.

  Both agents use our Model Context Protocol (MCP) Server to access drug 
  databases and local files securely.

  Finally, if the user asks to log a medication, the workflow pauses at the 
  Human-in-the-Loop gate to ask for your confirmation before writing any data."

* ACTION: (Pause for 3 seconds)


[2:00 — LIVE DEMO]

* VISUAL: (Switch screen to the running playground at http://localhost:18081)

* SPEAK:
  "Let's look at three quick examples in our live playground."


* ACTION: (Type and run Test Case 1 in the chat input)
  * INPUT: "My name is John Doe, my SSN is 123-45-6789, and my email is john@example.com. Can you tell me the guidelines for Metformin?"

* SPEAK:
  "Notice that the SSN and email are scrubbed instantly to protect privacy. 
  The safety agent then successfully fetches the Metformin guidelines."

* ACTION: (Pause for 2 seconds)


* ACTION: (Type and run Test Case 2 in the chat input)
  * INPUT: "I am taking Aspirin and Warfarin. Is this safe?"

* SPEAK:
  "Second, let's test drug interaction. The safety agent checks the database 
  and alerts us that combining these two medications is high-risk and poses a

* ACTION: (Pause for 2 seconds)


* ACTION: (Type and run Test Case 3 in the chat input)
  * INPUT: "Can you log that I took 1 pill of Metformin now?"

* SPEAK:
  "Third, let's log a dose. The scheduler agent stages the log entry, and the 
  workflow pauses. We get a prompt asking for our approval. 

  When we type 'yes', the system securely logs the dose in our care file."

* ACTION: (Pause for 2 seconds)
