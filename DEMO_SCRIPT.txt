[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through the architecture diagram to see how data flows.
First, user queries enter the Security Checkpoint node. This node filters out prompt injections and redacts sensitive PII like Social Security Numbers and email addresses.
Next, clean queries pass to the main Orchestrator Agent.
The Orchestrator utilizes AgentTools to delegate tasks to specialized sub-agents. It calls the safety_agent to check drug-drug interactions and guidelines, and the scheduler_agent to stage medication logs.
These agents communicate with our Model Context Protocol, or MCP, Server, which exposes tools to read and write database records.
Finally, the flow checks if a logging action requires human approval. If so, it routes to the hitl_approval node to pause and request permission before writing. Otherwise, it delivers the output directly to the final_output node.

[2:00 — LIVE DEMO]
(switch to playground at localhost:18081)
Now, let's look at three live scenarios.
In our first test case, we submit a query containing clinical questions alongside personal information: 'My name is John Doe, my SSN is 123-45-6789, and my email is john@example.com. Can you tell me the guidelines for Metformin?'
Notice that the SSN and email are redacted instantly by the Security Checkpoint before any data reaches the model. The safety_agent then successfully fetches the guidelines for Metformin.

(pause)

For our second case, we test drug interaction safety: 'I am taking Aspirin and Warfarin. Is this safe?'
The Orchestrator delegates this to the safety_agent, which queries the MCP server's check_drug_interactions tool and alerts us that this combination poses a severe bleeding hazard.

(pause)

In our third case, we test medication logging: 'Can you log that I took 1 pill of Metformin now?'
The scheduler_agent stages this action. The graph router detects the request, and the execution is paused at the hitl_approval gate. The user is prompted: '✋ Med-Companion requires your approval to perform the following action...'
Once we reply 'yes', the system writes the entry to our care logs file and completes the execution.

[3:00 — SECURITY & MCP]
This secure workflow ensures that patient queries are scrubbed of PII on entry, and that sensitive actions are never executed without human authorization. The MCP server serves as our clean database layer, exposing specific tool boundaries to each agent.
